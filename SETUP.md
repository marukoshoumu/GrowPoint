# グローポイント AI支援記録自動化 — セットアップ手順

## 前提条件

- Google Workspace アカウント
- Gemini API キー（Google AI Studio で取得）
- Google Drive の十分な空き容量

---

## 1. Google スプレッドシートの作成

1. Google Drive で新規スプレッドシートを作成
2. 名前を「グローポイント_支援記録管理」に変更
3. スプレッドシートのIDをメモ（URLの `/d/` と `/edit` の間の文字列）

## 2. GAS プロジェクトの作成

1. スプレッドシートのメニュー「拡張機能」→「Apps Script」を開く
2. 既存の `コード.gs` の内容を削除
3. 以下の順序で `.gs` ファイルを作成し、各ファイルの内容をコピー＆ペースト：

| ファイル名 | 役割 |
|---|---|
| `config.gs` | 設定値・API キー管理 |
| `utils.gs` | 共通ユーティリティ（ログ、API呼び出し、日付書式） |
| `prompts.gs` | AIプロンプトテンプレート |
| `fileManager.gs` | ファイル検知・移動 |
| `transcribe.gs` | Stage 1: 文字起こし |
| `extract.gs` | Stage 2: 構造化抽出 + バリデーション |
| `generateDocument.gs` | Stage 3: 統合ドキュメント生成（記録票+シート） |
| `templateManager.gs` | テンプレート差し込み |
| `dashboard.gs` | ダッシュボード・マスターデータ管理 |
| `main.gs` | エントリポイント・トリガー管理 |

## 3. Script Properties の設定

Apps Script エディタで「プロジェクトの設定」→「スクリプト プロパティ」を開き、以下を設定：

| プロパティ名 | 値 | 備考 |
|---|---|---|
| `GEMINI_API_KEY` | AIzaSy... | Google AI Studio で取得 |
| `SPREADSHEET_ID` | 手順1でメモしたID | |
| `TEMPLATE_ID_MONITORING_DOCUMENT` | （後で設定） | 計画モニタ統合テンプレートのドキュメントID |

> **注意**: フォルダIDは初回セットアップで自動設定されます。手動設定は不要です。

## 4. 初回セットアップの実行

1. スプレッドシートを再読み込み（ページリフレッシュ）
2. メニューバーに「グローポイント AI支援」が表示される
3. 「グローポイント AI支援」→「初回セットアップ」を実行
4. 権限の承認ダイアログが表示されたら許可する

これにより以下が自動作成されます：
- Google Drive にフォルダ構成（01_未処理 〜 06_エラー）
- スプレッドシートに4つのシート（処理状況、利用者マスター、用語集、処理ログ）
- 5分間隔の自動処理トリガー

## 5. Google Docs テンプレートの準備

### テンプレート_計画モニタ（統合テンプレート）

盛岡市の `keikakumonita.doc` と同じ構成で、**1つの Google Docs に3帳票**を含めます。

> `docs/` はリポジトリでは管理しない想定のため、指示文ファイルはローカルに用意したパス（または同等の資料）から開いてください。

1. `docs/templates/claude-desktop-template-instructions.md` の指示文を Claude Desktop にコピー＆ペーストして、テンプレートを自動作成してもらう
2. 作成されたファイルを Google Docs に変換・アップロード
3. テンプレートのドキュメントIDを `TEMPLATE_ID_MONITORING_DOCUMENT` に設定

テンプレートに含まれるプレースホルダ（約60個）:

```
【個別支援計画セクション（マスターデータから転記）】
{{service_type}}               サービスの種類
{{creation_date}}              計画作成年月日
{{consent_date}}               同意日
{{plan_section_1_needs}}       本人・家族の意向（ニーズ）
{{plan_section_4_notes}}       計画特記事項

【3帳票共通（マスターデータ）】
{{user_name}}                  利用者名
{{staff_name}}                 担当者名
{{date}}                       実施年月日
{{previous_monitoring_date}}   前回モニタリング実施日
{{next_monitoring_month}}      次回モニタリング予定月
{{long_term_goal}}             長期目標（計画・記録票で共有）
{{short_term_goal_1}}          短期目標①（計画・記録票で共有）
{{support_content_1}}          支援内容①
{{goal_1_period}}              期間①
{{short_term_goal_2}}          短期目標②
{{support_content_2}}          支援内容②
{{goal_2_period}}              期間②
{{attendees}}                  出席者

【記録票セクション（AI生成）】
{{section_1_intention}}        1. 本人・家族の意向
{{section_4_status_1}}         4.1 短期目標①に対する支援の実施状況
{{section_4_status_2}}         4.2 短期目標②に対する支援の実施状況
{{section_5_impression}}       5. 支援を受けた感想
{{section_6_future}}           6. 今後の支援方針
{{section_7_notes}}            7. 特記事項

【シートセクション（AI生成 + Phase 1でマスター連携）】
{{wl_note_1}} 〜 {{wl_note_10}}     職業生活 特記事項（10項目・要約）
{{rel_note_1}} 〜 {{rel_note_5}}    対人関係 特記事項（5項目）
{{task_note_1}} 〜 {{task_note_5}}  作業 特記事項（5項目）
{{overall_assessment}}              総合所見

職業生活・対人関係・作業の**評価（1/2/3）**は GAS で差し込まない。テンプレの各評価マスに **「1　2　3」** を固定で記載する（`{{wl_rating_*}}` 等のプレースホルダは使わない）。

【フッター（任意）】
{{ai_disclaimer}}              AI下書き注記（テンプレに無くても可。無い場合は GAS が渡す値は未使用）
```

原本（`keikakumonita.doc`）どおり、ヘッダー表の「サービス管理責任者」は**固定ラベル**とし、氏名用の `{{service_manager}}` は**置かない**。マスターのサービス管理責任者（利用者マスター C 列）はプロンプト文脈用。

- **記録票の AI 出力**は Stage 3-A が `## 4.1` / `## 4.2` 見出しで書く想定。旧テンプレの `{{section_4_status}}`（単一）だけでは GAS は置換しないため、**テンプレは `_1` / `_2` に更新すること**。
- **面談全体のまとめ**は `{{overall_assessment}}`（総合所見）。`{{ai_disclaimer}}` に要約を書かない。

> **ヒント**: テンプレートが未設定でもシステムは動作します。その場合はプレーンテキストファイルとして出力されます。

### Gemini でテスト用の1行を作る（タブ区切り・コピペ）

「利用者マスター」シートに**1行目のヘッダーが既にある**前提で、**2行目に貼るデータ行だけ**を Gemini に出させます。出力は**プレーンテキスト1行**、列の区切りは**タブ文字のみ**（カンマ不可）。

1. 下のプロンプトを Gemini に貼る（`利用者名` は音声ファイル名と揃える名前に書き換えてもよい）。
2. 応答から**データ行1行だけ**をコピーする（説明文やコードブロックに囲まれた場合は中身だけ）。
3. スプレッドシートの「利用者マスター」で **A2** を選択して貼り付け（列が19列に収まることを確認）。

```
あなたの出力は次の1行だけ。説明・見出し・マークダウン・コードフェンスは禁止。

次の19列を、この順序のまま、タブ区切り（TSV）の1行で出力してください。
列名は出力に含めない。データ値のみ1行。

列の意味（左から順）:
利用者名, 担当者名, サービス管理責任者, サービスの種類, 計画作成年月日, 同意日, 本人の意向, 長期目標, 短期目標①, 支援内容①, 期間①, 短期目標②, 支援内容②, 期間②, 計画特記事項, 前回モニタリング日, 次回モニタリング予定月, 前回の主な課題, 出席者

条件:
- 架空の就労継続支援B型の利用者。利用者名は「テスト花子」（音声ファイル名と一致させるためこの名前に固定）。
- 日付は2026年想定で整合させる（例: 2026年3月15日、2026年9月 など表記ゆれしにくい形）。
- サービスの種類は「就労継続支援B型」。
- セル内にタブ・改行は入れない。カンマ区切りにしない。
```

貼り付け後、**利用者名セル**と **`01_未処理` に上げる音声ファイル名**（拡張子除く）が一致しているか確認する。

## 7. 音声ファイルの命名規則

```
{利用者名}.m4a           例: 山田太郎.m4a
{利用者名}_{日付}.m4a     例: 山田太郎_2026-03-20.m4a
{日付}_{利用者名}.m4a     例: 2026-03-20_山田太郎.m4a
```

対応フォーマット: `.m4a`, `.mp3`, `.wav`, `.ogg`, `.webm`, `.aac`, `.flac`, `.mp4`

## 8. 日常の運用フロー

```
① 面談時   → スマホで録音
② 面談後   → 音声ファイルを「01_未処理」フォルダにアップロード
③ 自動処理 → 5分以内にパイプラインが自動起動
④ 翌朝     → スプレッドシートで「ドラフト完了」を確認
⑤ 確認     → ドキュメントリンクをクリックして内容を確認・修正（約5〜10分）
⑥ 完了     → 問題なければ承認済みにする
```

## トラブルシューティング

### メニューが表示されない
→ スプレッドシートをリロードしてください

### Drive にフォルダが作られない／手動削除後に再作成されない

**原因**: Drive 上でフォルダを消しても、**スクリプトプロパティ**に **古いフォルダ ID**（`FOLDER_ID_ROOT` など）が残っていると、コードは「既に作成済み」と判断し**新規作成しません**。

**対処**:

1. **最新の `main.gs` / `utils.gs` をデプロイ**したうえで「初回セットアップ」を再実行する（ルートが無効な ID なら自動でプロパティをクリアして再作成する処理が入っている）。
2. それでも直らないときは、Apps Script の **プロジェクトの設定 → スクリプトプロパティ**から、`FOLDER_ID_` で始まるキーを**すべて削除**してから、もう一度「初回セットアップ」を実行する。

### 「権限がありません」エラー
→ 初回実行時にスコープ承認が必要です。ダイアログに従って許可してください

### 処理がエラーになる
1. 処理状況シートの「エラー内容」列を確認
2. 処理ログシートで詳細を確認
3. よくある原因：
   - 利用者マスターに該当ユーザーがいない → ファイル名を確認
   - Gemini API エラー → API キーの有効性を確認
   - テンプレートエラー → テンプレートIDとプレースホルダを確認

### テンプレートなしで動作確認したい
→ `TEMPLATE_ID_MONITORING_DOCUMENT` を空のままにすると、プレーンテキストファイルとして出力されます。

### 「テンプレート確認」で NG になる／記録票とシートが別表示になる

**現行コード**では **計画モニタ用の統合テンプレ 1 本**だけを確認します（メニュー表示は **【計画モニタ（統合テンプレート）】**）。**【モニタリング記録票】【モニタリングシート】と2段表示される**場合は、Apps Script に **古い版のコード**が残っている可能性が高いです。**`gas/` 一式を最新に差し替え**（`main.gs`・`templateManager.gs`・`config.gs` など）して保存・再実行してください。

**「ドキュメントがありません。ドキュメントが削除されたか、読み取りアクセス権がない可能性があります。」** は、次のときに出やすいです。

- Script Properties の **`TEMPLATE_ID_MONITORING_DOCUMENT`** が、**別アカウントのドキュメント**を指している、または **削除済み・ID のコピーミス**
- テンプレ Google ドキュメントを、**スクリプトを実行している Google アカウント**と共有していない（閲覧可でよい）

対処: ブラウザで `https://docs.google.com/document/d/（ID部分）/edit` を開き、**同じアカウントで開けるか**確認する。Script Properties の ID は URL の `/d/` と `/edit` の間の文字列。

**NG: undefined** は、例外メッセージが取れない場合の表示残りです。コード側は `templateManager.gs` の `verifyTemplate` でエラー文言を補完するよう更新済み。**最新コードに更新**したうえで、上記 ID・共有を確認してください。

### トリガーを停止したい
→ メニュー「グローポイント AI支援」→「トリガー停止」
