# グローポイント AI支援記録自動化

Google Apps Script（GAS）で、面談音声から記録票・モニタリング関連ドキュメントまでを自動生成するソースです。処理は **Stage1（文字起こし）→ Stage2（構造化抽出）→ Stage3（ドキュメント生成）** のパイプラインで進みます。Stage1・2 のモデルは `gas/config.gs` の `STAGE1_MODEL` / `STAGE2_MODEL`（既定は `gemini-2.5-flash` 系）で定義されています。

## 前提条件

- Google アカウント（スプレッドシート・Drive・Apps Script）
- Gemini API キー（[Google AI Studio](https://aistudio.google.com/) などで取得）
- Google Drive の十分な空き容量

## リポジトリ構成

| パス | 内容 |
|------|------|
| `gas/` | `.gs` ソースと `appsscript.json` |
| `.clasp.json.example` | [clasp](https://github.com/google/clasp) 利用時の設定例（実ファイル `.clasp.json` はリポジトリに含めない） |

## コードの反映（どちらか）

### A. エディタに手動でコピー

1. スプレッドシートから「拡張機能」→「Apps Script」を開く。  
2. 既存の `コード.gs` があれば削除し、`gas/` 内の各 `.gs` と同じ名前のファイルを作成して内容を貼り付ける。  
3. `appsscript.json` の内容（タイムゾーン `Asia/Tokyo`、V8 など）をプロジェクト設定と整合させる。

ファイル例（すべて `gas/` 直下）: `config.gs`, `utils.gs`, `prompts.gs`, `fileManager.gs`, `transcribe.gs`, `extract.gs`, `generateDocument.gs`, `templateManager.gs`, `dashboard.gs`, `main.gs`。

### B. clasp で push

1. `.clasp.json.example` を `.clasp.json` にコピーし、`scriptId` を**自分の** Apps Script プロジェクト ID に差し替える。  
2. `clasp login` のあと `clasp push` などで `gas/` を反映する。

`.clasp.json` に API キーは含まれませんが、`scriptId` はプロジェクト固有のため **Git にはコミットせず**（`.gitignore` で除外）、チームごとにローカルで管理してください。

## 1. スプレッドシートの用意

1. Google Drive で新規スプレッドシートを作成する。  
2. URL の `/d/` と `/edit` の間の文字列が **スプレッドシート ID** になる。

## 2. Script Properties

Apps Script エディタで「プロジェクトの設定」→「スクリプト プロパティ」に次を設定する。

| プロパティ名 | 必須 | 説明 |
|--------------|------|------|
| `GEMINI_API_KEY` | はい | Gemini API キー |
| `SPREADSHEET_ID` | はい | 上記スプレッドシート ID |
| `TEMPLATE_ID_MONITORING_DOCUMENT` | いいえ | 計画モニタ用 **統合テンプレ** の Google ドキュメント ID。未設定の場合はプレーンテキスト等での出力にフォールバック |

フォルダ ID（`FOLDER_ID_*`）は **初回セットアップ** で自動設定される。Drive 上でフォルダを手で消したあと再作成したい場合は、古い `FOLDER_ID_*` が残っていると再作成されないことがある。そのときはトラブルシューティングを参照。

### 任意: プロンプトを Google ドキュメントで管理

次を設定すると、各 Stage のプロンプトを **ドキュメント本文** から読み込む（未設定または読み込み失敗時は `prompts.gs` 内の埋め込みプロンプトを使用）。

| プロパティ名 | 対象 |
|--------------|------|
| `PROMPT_FILE_ID_STAGE1` | Stage1 文字起こし |
| `PROMPT_FILE_ID_STAGE2` | Stage2 構造化抽出 |
| `PROMPT_FILE_ID_STAGE3A` | Stage3-A（モニタリング記録票の下書き本文） |
| `PROMPT_FILE_ID_STAGE3B` | Stage3-B（モニタリングシートの総合所見テキストのみ） |

## 3. 初回セットアップ

1. スプレッドシートを再読み込みする。  
2. メニュー **「グローポイント AI支援」** → **「初回セットアップ」** を実行し、権限を承認する。  

これにより Drive 上にフォルダ（`01_未処理` 〜 `06_エラー` など）、スプレッドシート上にシート（処理状況・利用者マスター・用語集・処理ログ）、**5 分間隔のトリガー** が作成される。

## 4. Google ドキュメントテンプレート

**計画モニタ用の統合テンプレ 1 本**を用意し、そのドキュメント ID を `TEMPLATE_ID_MONITORING_DOCUMENT` に設定する。プレースホルダの一覧・検証は、スプレッドシートのメニューから実行できる **テンプレート確認**（実装は `templateManager.gs`）を参照。

メニュー表示が **【計画モニタ（統合テンプレート）】** の 1 系統でない場合は、エディタに載っているコードが古い可能性があるため `gas/` 一式を最新に置き換える。

## 5. 利用者マスター

「利用者マスター」シートの 1 行目は初回セットアップでヘッダーが作成される。**利用者名（A 列）は音声ファイル名（拡張子除く）と一致**させる。

テスト用の 1 行をタブ区切りで作りたい場合は、列順はヘッダー行と同じ 19 列（利用者名〜出席者）に合わせ、説明文を出さず **データ行のみ** をスプレッドシートに貼り付ける。

## 6. 音声ファイル

命名の例: `{利用者名}.m4a`, `{利用者名}_{日付}.m4a`, `{日付}_{利用者名}.m4a`（実装は `parseFileNameForUser` などに準拠）。

取り込み可否は **Drive 上の MIME タイプ**で判定する（`gas/fileManager.gs` の `isAudioFile_`）。例: MPEG-4 音声・WAV・OGG・WebM・AAC・FLAC、およびスマホ録音で `video/mp4` になる場合がある。

## 7. 日常の運用の流れ

1. 面談を録音する。  
2. 音声を **「01_未処理」** にアップロードする。  
3. トリガーにより数分以内にパイプラインが動く。  
4. 「処理状況」シートでステータス・ドキュメントリンクを確認する。  
5. 内容を確認・修正し、問題なければ運用ルールに従って完了扱いにする。

## トラブルシューティング

- **メニューが出ない** → スプレッドシートを再読み込みする。  
- **Drive にフォルダが作られない / 消したあと再作成されない** → `FOLDER_ID_ROOT` が無効なら初回セットアップがクリアして再作成する処理がある。直らない場合はスクリプトプロパティから `FOLDER_ID_` で始まるキーを削除してから「初回セットアップ」を再実行。  
- **権限エラー** → 初回実行時に表示される承認を完了する。  
- **処理エラー** → 「処理状況」のエラー列、「処理ログ」シートを確認。利用者マスター不一致・API キー・テンプレ ID・共有設定が多い。  
- **`TEMPLATE_ID_MONITORING_DOCUMENT` 関連の「ドキュメントがありません」** → ID の誤り、削除済みドキュメント、またはスクリプト実行アカウントとテンプレの共有がない場合に起きやすい。ブラウザで `https://docs.google.com/document/d/（ID）/edit` が開けるか確認する。  
- **トリガーを止めたい** → メニュー「グローポイント AI支援」→「トリガー停止」（実装は `main.gs` の `removeTrigger`）。
